- Content byte comparison replaces CID comparison (fixes false-positive changes with encryption)
- New file: `src/utils/contentfetcher.ts`

### SyncEngine integration tests
- In-memory UCN clock server via `@storacha/ucn/server` — no remote service needed
- Test infra: SharedNetwork (simulates Storacha), per-peer blockstores (local + shared fallback), identity encoder (raw hash), mock StorageClient (CAR → shared blocks)
- `SyncDeps.remotes?: ClockConnection[]` threaded through all resolve/publish calls
- 12 tests: single/multi-file sync, peer propagation, update, delete, dedup, stop, markdown, nested dirs, pullRemote
- **Bug found:** `diffRemoteChanges` didn't detect deletions (paths in before but not after) — fixed

### Key learning: UCN concurrent publish
- Publishing a revision based on stale state while server has newer state causes hang
- Must resolve to latest state before building revision — this matches real sync flow
- First v0Put bootstraps the name; subsequent puts must build on resolved state
